# Sistema de RH para o Mercado Brasileiro

# Regras de Negócio (RN)

## Bloco 20 – LGPD, Segurança e Governança

### Versão

1.0

### Objetivo

Este documento descreve as regras de negócio relacionadas à proteção de dados pessoais, segurança da informação, governança de acessos, auditoria, conformidade regulatória, retenção de dados e gestão de riscos.

---

# Governança de Dados

## RN-2301

O sistema deve possuir política de governança de dados configurável.

## RN-2302

O sistema deve permitir classificação dos dados armazenados.

## RN-2303

O sistema deve identificar dados pessoais.

## RN-2304

O sistema deve identificar dados pessoais sensíveis.

## RN-2305

O sistema deve permitir definição de responsáveis pelos dados.

## RN-2306

O sistema deve registrar origem dos dados.

## RN-2307

O sistema deve registrar finalidade do tratamento.

## RN-2308

O sistema deve registrar base legal aplicável.

## RN-2309

O sistema deve manter histórico das classificações.

## RN-2310

Toda movimentação deve ser auditável.

---

# Consentimento e Base Legal

## RN-2311

O sistema deve registrar consentimentos quando aplicável.

## RN-2312

O sistema deve registrar data e hora do consentimento.

## RN-2313

O sistema deve registrar finalidade associada.

## RN-2314

O sistema deve permitir revogação do consentimento.

## RN-2315

A revogação deve ser registrada historicamente.

## RN-2316

O sistema deve registrar outras bases legais de tratamento.

## RN-2317

O sistema deve permitir consulta dos registros.

## RN-2318

O sistema deve impedir tratamento incompatível com a finalidade registrada.

## RN-2319

Toda alteração deve ser rastreável.

## RN-2320

Toda movimentação deve ser auditável.
