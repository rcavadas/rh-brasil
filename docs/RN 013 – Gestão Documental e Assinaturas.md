# Sistema de RH para o Mercado Brasileiro

# Regras de Negócio (RN)

## Bloco 13 – Gestão Documental e Assinaturas

### Versão

1.0

### Objetivo

Este documento descreve as regras de negócio relacionadas à gestão eletrônica de documentos (GED), armazenamento documental, versionamento, retenção, assinatura eletrônica, assinatura digital ICP-Brasil e auditoria documental.

---

# Estrutura Documental

## RN-1551

O sistema deve possuir repositório centralizado de documentos.

## RN-1552

Todo documento deve possuir identificador único.

## RN-1553

Todo documento deve possuir categoria documental.

## RN-1554

Todo documento deve possuir data de criação.

## RN-1555

Todo documento deve possuir responsável pelo envio.

## RN-1556

O sistema deve permitir classificação documental.

## RN-1557

O sistema deve permitir organização por empresa.

## RN-1558

O sistema deve permitir organização por colaborador.

## RN-1559

O sistema deve permitir organização por processo.

## RN-1560

Toda movimentação documental deve ser auditável.

---

# Tipos de Documentos

## RN-1561

O sistema deve permitir cadastro de tipos documentais.

## RN-1562

O sistema deve permitir documentos admissionais.

## RN-1563

O sistema deve permitir contratos de trabalho.

## RN-1564

O sistema deve permitir termos aditivos.

## RN-1565

O sistema deve permitir documentos de férias.

## RN-1566

O sistema deve permitir documentos rescisórios.

## RN-1567

O sistema deve permitir ASOs.

## RN-1568

O sistema deve permitir certificados de treinamento.

## RN-1569

O sistema deve permitir documentos personalizados.

## RN-1570

Toda categoria documental deve possuir configuração própria.

---

# Documentos do Colaborador

## RN-1571

Cada colaborador deve possuir prontuário eletrônico.

## RN-1572

O prontuário deve centralizar documentos vinculados.

## RN-1573

O sistema deve permitir consulta cronológica.

## RN-1574

O sistema deve permitir filtros documentais.

## RN-1575

O sistema deve permitir anexação manual.

## RN-1576

O sistema deve permitir geração automática de documentos.

## RN-1577

O sistema deve manter histórico documental completo.

## RN-1578

O sistema deve permitir visualização por perfil autorizado.

## RN-1579

O sistema deve registrar acessos ao prontuário.

## RN-1580

Toda movimentação deve ser auditável.

---

# Upload e Armazenamento

## RN-1581

O sistema deve permitir upload de documentos.

## RN-1582

O sistema deve validar formatos permitidos.

## RN-1583

O sistema deve validar tamanho máximo configurado.

## RN-1584

O sistema deve realizar verificação de integridade do arquivo.

## RN-1585

O sistema deve registrar data e hora do upload.

## RN-1586

O sistema deve registrar usuário responsável.

## RN-1587

O sistema deve impedir arquivos corrompidos.

## RN-1588

O sistema deve manter histórico de substituições.

## RN-1589

O sistema deve registrar origem do documento.

## RN-1590

Todo armazenamento deve possuir rastreabilidade.

---

# Versionamento

## RN-1591

O sistema deve possuir controle de versões.

## RN-1592

Toda nova versão deve preservar versões anteriores.

## RN-1593

O sistema deve identificar versão vigente.

## RN-1594

O sistema deve registrar responsável pela alteração.

## RN-1595

O sistema deve registrar motivo da alteração.

## RN-1596

O sistema deve permitir comparação entre versões.

## RN-1597

O sistema deve impedir perda de histórico.

## RN-1598

O sistema deve registrar data e hora das alterações.

## RN-1599

Toda versão deve possuir rastreabilidade.

## RN-1600

Todo versionamento deve ser auditável.

---

# Geração Automática de Documentos

## RN-1601

O sistema deve permitir geração automática de documentos.

## RN-1602

O sistema deve utilizar modelos parametrizáveis.

## RN-1603

O sistema deve permitir campos dinâmicos.

## RN-1604

O sistema deve permitir geração de contratos.

## RN-1605

O sistema deve permitir geração de recibos.

## RN-1606

O sistema deve permitir geração de avisos de férias.

## RN-1607

O sistema deve permitir geração de documentos rescisórios.

## RN-1608

O sistema deve permitir geração em lote.

## RN-1609

O sistema deve registrar origem da geração.

## RN-1610

Toda geração deve ser auditável.

---

# Assinatura Eletrônica

## RN-1611

O sistema deve permitir assinatura eletrônica de documentos.

## RN-1612

A assinatura deve registrar identidade do signatário.

## RN-1613

A assinatura deve registrar data e hora.

## RN-1614

A assinatura deve registrar endereço IP quando aplicável.

## RN-1615

A assinatura deve registrar dispositivo utilizado quando possível.

## RN-1616

O sistema deve registrar evidências da assinatura.

## RN-1617

O sistema deve permitir múltiplos signatários.

## RN-1618

O sistema deve controlar ordem de assinatura quando configurada.

## RN-1619

Documentos assinados devem ser protegidos contra alterações.

## RN-1620

Toda assinatura deve ser auditável.

---

# Assinatura Digital ICP-Brasil

## RN-1621

O sistema deve suportar assinatura digital ICP-Brasil.

## RN-1622

O sistema deve validar certificados digitais utilizados.

## RN-1623

O sistema deve registrar cadeia de certificação.

## RN-1624

O sistema deve registrar carimbo de tempo quando aplicável.

## RN-1625

O sistema deve armazenar evidências criptográficas.

## RN-1626

O sistema deve permitir múltiplas assinaturas digitais.

## RN-1627

O sistema deve preservar validade jurídica dos documentos.

## RN-1628

O sistema deve registrar histórico de assinaturas digitais.

## RN-1629

Toda assinatura ICP-Brasil deve possuir rastreabilidade completa.

## RN-1630

Toda operação deve ser auditável.

---

# Retenção Documental

## RN-1631

O sistema deve permitir configuração de políticas de retenção.

## RN-1632

As políticas podem variar por tipo documental.

## RN-1633

As políticas devem considerar requisitos legais.

## RN-1634

O sistema deve controlar prazo de retenção.

## RN-1635

O sistema deve alertar sobre documentos próximos do vencimento.

## RN-1636

O sistema deve controlar descarte documental quando permitido.

## RN-1637

O descarte deve exigir autorização.

## RN-1638

O descarte deve manter registro histórico.

## RN-1639

O sistema deve impedir descarte indevido.

## RN-1640

Toda movimentação deve ser auditável.

---

# Pesquisa e Consulta

## RN-1641

O sistema deve permitir pesquisa textual.

## RN-1642

O sistema deve permitir filtros por colaborador.

## RN-1643

O sistema deve permitir filtros por tipo documental.

## RN-1644

O sistema deve permitir filtros por período.

## RN-1645

O sistema deve permitir filtros por status de assinatura.

## RN-1646

O sistema deve permitir pesquisa por identificador.

## RN-1647

O sistema deve permitir exportação autorizada.

## RN-1648

O sistema deve registrar consultas realizadas.

## RN-1649

Toda exportação deve respeitar permissões de acesso.

## RN-1650

Toda consulta documental deve possuir trilha de auditoria.
