# Sistema de RH para o Mercado Brasileiro

# Regras de Negócio (RN)

## Bloco 08 – SST, Medicina Ocupacional e Segurança do Trabalho

### Versão

1.0

### Objetivo

Este documento descreve as regras de negócio relacionadas à Saúde e Segurança do Trabalho (SST), medicina ocupacional, gestão de riscos, exames ocupacionais, CAT, EPI, treinamentos obrigatórios e integração com os eventos SST do eSocial.

---

# Estrutura de SST

## RN-791

Toda empresa deve possuir estrutura de SST parametrizada.

## RN-792

O sistema deve permitir cadastro de unidades de trabalho.

## RN-793

O sistema deve permitir cadastro de setores.

## RN-794

O sistema deve permitir cadastro de ambientes de trabalho.

## RN-795

Todo ambiente deve possuir vigência.

## RN-796

Alterações em ambientes devem manter histórico.

## RN-797

O sistema deve permitir associação entre colaborador e ambiente.

## RN-798

O sistema deve registrar movimentações entre ambientes.

## RN-799

Toda movimentação deve possuir data de vigência.

## RN-800

Toda alteração deve ser auditável.

---

# Gestão de Riscos Ocupacionais

## RN-801

O sistema deve permitir cadastro de fatores de risco.

## RN-802

Os riscos devem ser classificados conforme legislação aplicável.

## RN-803

O sistema deve permitir riscos físicos.

## RN-804

O sistema deve permitir riscos químicos.

## RN-805

O sistema deve permitir riscos biológicos.

## RN-806

O sistema deve permitir riscos ergonômicos.

## RN-807

O sistema deve permitir riscos de acidentes.

## RN-808

Todo risco deve possuir vigência.

## RN-809

O sistema deve permitir vinculação de riscos aos ambientes.

## RN-810

O sistema deve manter histórico das exposições.

# Exames Ocupacionais

## RN-841

O sistema deve controlar exames admissionais.

## RN-842

O sistema deve controlar exames periódicos.

## RN-843

O sistema deve controlar exames de retorno ao trabalho.

## RN-844

O sistema deve controlar exames de mudança de função.

## RN-845

O sistema deve controlar exames demissionais.

## RN-846

Todo exame deve possuir data prevista.

## RN-847

Todo exame deve possuir data de realização.

## RN-848

Todo exame deve possuir resultado registrado.

## RN-849

# ASO – Atestado de Saúde Ocupacional

## RN-851

Todo exame ocupacional deve permitir emissão de ASO.

## RN-852

O ASO deve possuir resultado apto ou inapto.

## RN-853

O ASO deve possuir data de emissão.

## RN-854

O sistema deve armazenar histórico de ASOs.

## RN-855

O sistema deve permitir anexação do documento assinado.

## RN-856

O sistema deve controlar vigência dos ASOs.

## RN-857

O sistema deve permitir consulta histórica.

## RN-858

O sistema deve registrar responsável pela emissão.

## RN-859

Toda alteração deve manter histórico.

## RN-860

O ASO deve possuir rastreabilidade completa.

# CAT – Comunicação de Acidente de Trabalho

## RN-861

## RN-862

## RN-863

## RN-864

O sistema deve registrar local da ocorrência.

## RN-865

## RN-866

O sistema deve permitir anexação de evidências.

## RN-867

O sistema deve permitir emissão da CAT.

## RN-868

## RN-869

O sistema deve controlar status da CAT.

## RN-870

# eSocial SST

## RN-891

O sistema deve gerar evento S-2210 quando aplicável.

## RN-892

O sistema deve gerar evento S-2220 quando aplicável.

## RN-893

O sistema deve gerar evento S-2240 quando aplicável.

## RN-894

O sistema deve validar dados obrigatórios antes da transmissão.

## RN-895

Eventos rejeitados devem permanecer pendentes de correção.

## RN-896

O sistema deve registrar motivo da rejeição.

## RN-897

Eventos corrigidos devem permitir reenvio.

## RN-898

O recibo retornado pelo eSocial deve ser armazenado.

## RN-899

O sistema deve permitir conciliação dos eventos transmitidos.

## RN-900

Toda transmissão SST deve possuir trilha completa de auditoria.
