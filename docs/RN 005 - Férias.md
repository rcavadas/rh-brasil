# Sistema de RH para o Mercado Brasileiro

# Regras de Negócio (RN)

## Bloco 05 – Férias

### Versão

1.0

### Objetivo

Este documento descreve as regras de negócio relacionadas à aquisição, concessão, programação, cálculo, pagamento, fracionamento e integração das férias com folha de pagamento e eSocial.

---

# Período Aquisitivo

## RN-411

Todo colaborador contratado sob regime CLT deve possuir período aquisitivo de férias.

## RN-412

O período aquisitivo deve iniciar na data de admissão.

## RN-413

Cada período aquisitivo deve possuir duração de 12 meses.

## RN-414

Ao final de um período aquisitivo deve ser iniciado automaticamente um novo período.

## RN-415

O sistema deve manter histórico de todos os períodos aquisitivos.

## RN-416

Períodos aquisitivos não podem ser excluídos após sua criação.

## RN-417

O sistema deve identificar automaticamente períodos aquisitivos em aberto.

## RN-418

O sistema deve controlar saldo de dias adquiridos.

## RN-419

O saldo de férias deve ser atualizado conforme eventos legais.

## RN-420

Toda movimentação deve possuir rastreabilidade.

---

# Faltas e Direito às Férias

## RN-421

O sistema deve controlar faltas injustificadas ocorridas durante o período aquisitivo.

## RN-422

As faltas injustificadas devem impactar a quantidade de dias de férias conforme legislação vigente.

## RN-423

O cálculo da redução de férias deve seguir a tabela legal aplicável.

## RN-424

Faltas justificadas não devem reduzir o direito às férias.

## RN-425

O sistema deve registrar memória de cálculo da redução.

## RN-426

A redução deve ser recalculada sempre que houver alteração das faltas.

## RN-427

O sistema deve manter histórico das alterações.

## RN-428

A apuração deve ocorrer por período aquisitivo.

## RN-429

O sistema deve permitir auditoria da apuração.

## RN-430

O sistema deve registrar origem das faltas consideradas.

---

# Período Concessivo

## RN-431

Todo período aquisitivo deve gerar um período concessivo correspondente.

## RN-432

O período concessivo deve possuir duração de 12 meses.

## RN-433

O sistema deve controlar vencimento do período concessivo.

## RN-434

O sistema deve alertar férias próximas do vencimento.

## RN-435

O sistema deve alertar férias vencidas.

## RN-436

O sistema deve permitir relatórios de férias vencidas.

## RN-437

O sistema deve registrar histórico de alertas emitidos.

## RN-438

O sistema deve permitir parametrização de antecedência dos alertas.

## RN-439

O sistema deve impedir perda de rastreabilidade do período concessivo.

## RN-440

Toda alteração deve ser auditável.

---

# Programação de Férias

## RN-441

O sistema deve permitir solicitação de férias pelo colaborador.

## RN-442

O sistema deve permitir programação de férias pelo RH.

## RN-443

O sistema deve permitir aprovação por gestor.

## RN-444

O sistema deve permitir fluxo de aprovação multinível.

## RN-445

A programação deve respeitar saldo disponível.

## RN-446

O sistema deve impedir programação superior ao saldo disponível.

## RN-447

O sistema deve controlar conflitos de datas.

## RN-448

O sistema deve registrar justificativas para reprovações.

## RN-449

Toda aprovação deve possuir responsável identificado.

## RN-450

Toda programação deve possuir histórico completo.

---

# Fracionamento

## RN-451

O sistema deve permitir fracionamento de férias quando permitido pela legislação.

## RN-452

O fracionamento deve respeitar a quantidade máxima de períodos permitidos.

## RN-453

Um dos períodos deve respeitar a duração mínima exigida pela legislação vigente.

## RN-454

Os demais períodos devem respeitar os limites mínimos legais.

## RN-455

O sistema deve validar automaticamente as regras de fracionamento.

## RN-456

Fracionamentos inválidos devem ser bloqueados.

## RN-457

O sistema deve registrar histórico dos períodos concedidos.

## RN-458

O sistema deve recalcular saldo após cada concessão.

## RN-459

O sistema deve controlar férias parciais.

## RN-460

O sistema deve impedir duplicidade de utilização dos dias.

---

# Abono Pecuniário

## RN-461

O colaborador pode solicitar conversão de parte das férias em abono pecuniário quando permitido pela legislação.

## RN-462

O sistema deve controlar prazo para solicitação.

## RN-463

Solicitações fora do prazo devem gerar alerta.

## RN-464

O sistema deve calcular automaticamente os dias convertidos.

## RN-465

O sistema deve calcular automaticamente o valor do abono.

## RN-466

O cálculo deve possuir memória detalhada.

## RN-467

O sistema deve integrar o valor à folha de férias.

## RN-468

O sistema deve registrar aprovação quando exigida.

## RN-469

O sistema deve registrar histórico da solicitação.

## RN-470

Toda movimentação deve ser auditável.

---

# Cálculo de Férias

## RN-471

O sistema deve calcular remuneração de férias conforme legislação vigente.

## RN-472

O sistema deve calcular adicional constitucional de um terço.

## RN-473

O cálculo deve considerar salário vigente na data da concessão.

## RN-474

O sistema deve considerar médias de verbas variáveis quando aplicável.

## RN-475

O sistema deve considerar médias de horas extras quando aplicável.

## RN-476

O sistema deve considerar médias de adicionais quando aplicável.

## RN-477

O sistema deve gerar memória detalhada de cálculo.

## RN-478

Diferenças identificadas devem permitir recálculo.

## RN-479

O cálculo deve integrar automaticamente à folha.

## RN-480

Todo cálculo deve ser auditável.

---

# Pagamento das Férias

## RN-481

O pagamento das férias deve respeitar o prazo legal vigente.

## RN-482

O sistema deve gerar aviso de férias.

## RN-483

O aviso deve possuir protocolo de emissão.

## RN-484

O sistema deve registrar ciência do colaborador quando aplicável.

## RN-485

O pagamento deve gerar recibo de férias.

## RN-486

O recibo deve permanecer disponível para consulta.

## RN-487

O sistema deve registrar data efetiva de pagamento.

## RN-488

O sistema deve controlar pagamentos pendentes.

## RN-489

Pagamentos não realizados dentro do prazo devem gerar alerta.

## RN-490

Toda movimentação financeira deve possuir rastreabilidade.

---

# Férias Coletivas

## RN-491

O sistema deve permitir programação de férias coletivas.

## RN-492

Férias coletivas podem ser aplicadas por empresa, unidade ou grupo de colaboradores.

## RN-493

O sistema deve controlar períodos coletivos.

## RN-494

O sistema deve recalcular saldos individuais após concessão coletiva.

## RN-495

O sistema deve manter histórico das férias coletivas.

## RN-496

O sistema deve permitir comunicação em massa aos colaboradores.

## RN-497

Toda concessão coletiva deve ser auditável.

## RN-498

O sistema deve gerar relatórios específicos de férias coletivas.

## RN-499

O sistema deve permitir integração com folha de pagamento.

## RN-500

O sistema deve permitir integração com eventos obrigatórios do eSocial.
