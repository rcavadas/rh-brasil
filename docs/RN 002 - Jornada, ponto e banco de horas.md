# Sistema de RH para o Mercado Brasileiro

# Regras de Negócio (RN)

## Bloco 02 – Jornada, Controle de Ponto e Banco de Horas

### Versão

1.0

### Objetivo

Este documento descreve as regras de negócio relacionadas ao controle de jornada, registro de ponto, banco de horas, horas extras, faltas, atrasos, adicional noturno e conformidade com a Portaria MTP nº 671/2021.

---

# Jornada de Trabalho

## RN-051

Todo colaborador sujeito a controle de jornada deve possuir jornada cadastrada.

## RN-052

A jornada deve possuir horário de entrada e saída definidos.

## RN-053

A jornada deve possuir intervalo intrajornada configurado.

## RN-054

A jornada deve possuir carga horária diária.

## RN-055

A jornada deve possuir carga horária semanal.

## RN-056

A jornada deve possuir data de vigência.

## RN-057

Alterações de jornada devem manter histórico.

## RN-058

Nenhuma alteração de jornada poderá sobrescrever registros anteriores.

## RN-059

O sistema deve permitir múltiplas jornadas para o mesmo colaborador ao longo do tempo.

## RN-060

A jornada aplicada deve ser aquela vigente na data da marcação.

---

# Escalas de Trabalho

## RN-061

O sistema deve permitir escalas fixas.

## RN-062

O sistema deve permitir escalas rotativas.

## RN-063

O sistema deve permitir escala 12x36.

## RN-064

O sistema deve permitir escala 6x1.

## RN-065

O sistema deve permitir escala 5x2.

## RN-066

O sistema deve permitir escalas personalizadas.

## RN-067

A escala deve possuir calendário de vigência.

## RN-068

O sistema deve permitir troca de escala mediante autorização.

## RN-069

Trocas de escala devem manter histórico.

## RN-070

O sistema deve registrar responsável pela aprovação da troca.

---

# Registro de Ponto

## RN-071

Toda marcação deve registrar data e hora exatas.

## RN-072

Toda marcação deve possuir identificador único.

## RN-073

O sistema deve impedir alteração direta da marcação original.

## RN-074

A marcação original deve permanecer armazenada permanentemente.

## RN-075

Correções devem gerar registros complementares.

## RN-076

Toda correção deve possuir justificativa.

## RN-077

Toda correção deve registrar usuário responsável.

## RN-078

O sistema deve registrar origem da marcação.

## RN-079

A origem poderá ser REP-C, REP-A, REP-P, Web ou Mobile.

## RN-080

Toda marcação deve possuir trilha de auditoria.

---

# Portaria MTP nº 671/2021

## RN-081

O sistema deve manter registros de ponto íntegros.

## RN-082

O sistema deve garantir rastreabilidade das marcações.

## RN-083

O sistema deve manter histórico completo de alterações.

## RN-084

O sistema deve disponibilizar espelho de ponto ao trabalhador.

## RN-085

O sistema deve permitir exportação dos registros.

## RN-086

O sistema deve armazenar registros pelo prazo legal.

## RN-087

O sistema deve registrar identificador do equipamento utilizado.

## RN-088

O sistema deve permitir emissão de comprovante de registro.

## RN-089

O comprovante deve possuir data e hora da marcação.

## RN-090

O comprovante deve permanecer disponível para consulta.

---

# Geolocalização e Evidências

## RN-091

O sistema poderá registrar coordenadas geográficas.

## RN-092

A geolocalização deve ser armazenada com a marcação.

## RN-093

O sistema poderá registrar evidência fotográfica.

## RN-094

As evidências devem respeitar requisitos da LGPD.

## RN-095

Imagens não poderão ser excluídas sem rastreabilidade.

---

# Tratamento de Jornada

## RN-096

O sistema deve identificar atrasos automaticamente.

## RN-097

O sistema deve identificar saídas antecipadas.

## RN-098

O sistema deve identificar faltas.

## RN-099

O sistema deve identificar ausências justificadas.

## RN-100

O sistema deve identificar ausências injustificadas.

---

# Horas Extras

## RN-101

Horas trabalhadas além da jornada devem ser classificadas como horas extras.

## RN-102

A classificação deve respeitar acordos coletivos vigentes.

## RN-103

O sistema deve permitir múltiplos percentuais de adicional.

## RN-104

O sistema deve suportar adicional de 50%.

## RN-105

O sistema deve suportar adicional de 100%.

## RN-106

O sistema deve permitir percentuais customizados.

## RN-107

O cálculo deve considerar feriados.

## RN-108

O cálculo deve considerar domingos.

## RN-109

O cálculo deve considerar jornadas especiais.

## RN-110

Horas extras devem alimentar automaticamente a folha de pagamento.

---

# Banco de Horas

## RN-111

O sistema deve permitir banco de horas individual.

## RN-112

O sistema deve permitir banco de horas coletivo.

## RN-113

O sistema deve registrar créditos de horas.

## RN-114

O sistema deve registrar débitos de horas.

## RN-115

Toda movimentação deve possuir data de lançamento.

## RN-116

O sistema deve calcular saldo atualizado.

## RN-117

O sistema deve controlar vencimento de horas.

## RN-118

Horas vencidas devem seguir política definida pela empresa ou acordo coletivo.

## RN-119

O colaborador deve poder consultar seu saldo.

## RN-120

O gestor deve poder consultar saldos de sua equipe.

---

# Adicional Noturno

## RN-121

O sistema deve identificar horas trabalhadas em período noturno.

## RN-122

O período noturno deve ser parametrizável conforme categoria.

## RN-123

A hora noturna reduzida deve ser considerada quando aplicável.

## RN-124

O adicional noturno deve ser calculado automaticamente.

## RN-125

O cálculo deve ser enviado para a folha de pagamento.

---

# Aprovações

## RN-126

Solicitações de ajuste devem exigir justificativa.

## RN-127

Solicitações devem possuir fluxo de aprovação.

## RN-128

Gestores devem poder aprovar ou rejeitar solicitações.

## RN-129

Toda decisão deve ser registrada.

## RN-130

O sistema deve registrar data e responsável pela decisão.

---

# Auditoria

## RN-131

Toda marcação deve possuir rastreabilidade completa.

## RN-132

Toda alteração deve gerar log.

## RN-133

Nenhuma alteração poderá excluir histórico anterior.

## RN-134

O sistema deve registrar endereço IP quando aplicável.

## RN-135

O sistema deve registrar dispositivo utilizado.

## RN-136

O sistema deve permitir auditoria completa das jornadas.

## RN-137

Os registros devem estar disponíveis para fiscalização trabalhista.

## RN-138

Os registros devem possuir integridade verificável.

## RN-139

Toda exportação deve ser registrada em log.

## RN-140

O sistema deve manter trilha completa de auditoria.
