# Sistema de RH para o Mercado Brasileiro

# Regras de Negócio (RN)

## Bloco 04 – Benefícios

### Versão

1.0

### Objetivo

Este documento descreve as regras de negócio relacionadas à administração de benefícios corporativos, elegibilidade, concessão, suspensão, integração com fornecedores, descontos em folha e auditoria.

---

# Estrutura de Benefícios

## RN-286

Todo benefício deve possuir cadastro próprio.

## RN-287

Todo benefício deve possuir código interno único.

## RN-288

Todo benefício deve possuir descrição.

## RN-289

Todo benefício deve possuir categoria definida.

## RN-290

O sistema deve permitir ativação e inativação de benefícios.

---

# Elegibilidade

## RN-291

A elegibilidade de benefícios deve ser parametrizável.

## RN-292

A elegibilidade pode considerar cargo.

## RN-293

A elegibilidade pode considerar função.

## RN-294

A elegibilidade pode considerar unidade organizacional.

## RN-295

A elegibilidade pode considerar centro de custo.

## RN-296

A elegibilidade pode considerar tipo de contrato.

## RN-297

A elegibilidade pode considerar sindicato.

## RN-298

A elegibilidade pode considerar tempo mínimo de empresa.

## RN-299

A elegibilidade deve possuir data de início de vigência.

## RN-300

Toda alteração de elegibilidade deve manter histórico.

---

# Vale-Transporte

## RN-301

O vale-transporte deve permitir múltiplos trajetos.

## RN-302

O colaborador deve informar origem e destino.

## RN-303

O sistema deve permitir validação dos trajetos cadastrados.

## RN-304

O benefício deve considerar dias efetivamente trabalhados.

## RN-305

A concessão deve considerar jornadas híbridas quando aplicável.

## RN-306

A concessão deve considerar jornadas remotas quando aplicável.

## RN-307

O desconto em folha deve respeitar a legislação vigente.

## RN-308

O sistema deve permitir concessão superior ao mínimo legal.

## RN-309

O sistema deve permitir suspensão temporária do benefício.

## RN-310

Toda alteração deve gerar histórico.

---

# Vale-Refeição

## RN-311

O sistema deve permitir concessão diária.

## RN-312

O sistema deve permitir concessão mensal.

## RN-313

O valor concedido deve ser parametrizável.

## RN-314

O valor pode variar conforme cargo ou política corporativa.

## RN-315

O sistema deve permitir suspensão temporária.

## RN-316

Afastamentos podem impactar o benefício conforme política definida.

## RN-317

Férias podem suspender concessões conforme parametrização.

## RN-318

O sistema deve registrar histórico de créditos.

## RN-319

O sistema deve permitir desconto de participação do colaborador.

## RN-320

Toda movimentação deve possuir rastreabilidade.

---

# Vale-Alimentação

## RN-321

O sistema deve permitir concessão periódica.

## RN-322

O valor deve ser parametrizável.

## RN-323

O benefício pode variar por unidade organizacional.

## RN-324

O benefício pode variar por acordo coletivo.

## RN-325

O sistema deve permitir participação financeira do colaborador.

## RN-326

O sistema deve controlar vigência das concessões.

## RN-327

O sistema deve controlar suspensões temporárias.

## RN-328

O sistema deve registrar histórico completo das concessões.

## RN-329

O sistema deve permitir integração com fornecedores externos.

## RN-330

Toda alteração deve possuir trilha de auditoria.

---

# Plano de Saúde

## RN-331

O sistema deve permitir múltiplas operadoras.

## RN-332

O sistema deve permitir múltiplos planos por operadora.

## RN-333

O colaborador deve poder possuir dependentes vinculados ao plano.

## RN-334

Dependentes devem possuir vigência própria.

## RN-335

Inclusões devem respeitar regras da operadora.

## RN-336

Exclusões devem respeitar regras da operadora.

## RN-337

O sistema deve controlar movimentações cadastrais.

## RN-338

O sistema deve controlar alterações de plano.

## RN-339

O sistema deve registrar histórico de movimentações.

## RN-340

Toda movimentação deve possuir responsável identificado.

---

# Coparticipação

## RN-341

O sistema deve permitir importação de eventos de coparticipação.

## RN-342

A coparticipação deve ser vinculada ao colaborador.

## RN-343

A coparticipação pode ser vinculada a dependentes.

## RN-344

O sistema deve permitir parcelamento de valores.

## RN-345

Parcelamentos devem manter histórico.

## RN-346

O sistema deve permitir limites máximos de desconto.

## RN-347

Descontos devem ser integrados à folha de pagamento.

## RN-348

O sistema deve registrar memória de cálculo.

## RN-349

Diferenças identificadas devem gerar pendências de conferência.

## RN-350

Toda importação deve possuir rastreabilidade.

---

# Plano Odontológico

## RN-351

O sistema deve permitir gestão independente do plano de saúde.

## RN-352

O colaborador pode possuir plano odontológico sem plano médico.

## RN-353

Dependentes devem possuir controle de vigência.

## RN-354

Movimentações devem ser registradas.

## RN-355

O sistema deve permitir integração com operadoras.

## RN-356

O sistema deve controlar descontos em folha.

## RN-357

O sistema deve registrar histórico de alterações.

## RN-358

O sistema deve registrar inclusões e exclusões.

## RN-359

Toda movimentação deve possuir responsável identificado.

## RN-360

O sistema deve permitir auditoria completa.

---

# Seguro de Vida

## RN-361

O sistema deve permitir contratação de seguro de vida corporativo.

## RN-362

O sistema deve controlar capitais segurados.

## RN-363

O sistema deve controlar beneficiários.

## RN-364

Beneficiários devem possuir vigência.

## RN-365

Alterações de beneficiários devem manter histórico.

## RN-366

O sistema deve controlar movimentações junto à seguradora.

## RN-367

O sistema deve permitir integração com fornecedores externos.

## RN-368

O sistema deve registrar inclusões automáticas quando previsto.

## RN-369

O sistema deve registrar exclusões automáticas quando previsto.

## RN-370

Toda movimentação deve ser auditável.

---

# Benefícios Flexíveis

## RN-371

O sistema deve permitir modelo de benefícios flexíveis.

## RN-372

O sistema deve permitir carteira de créditos.

## RN-373

O colaborador deve poder escolher benefícios elegíveis.

## RN-374

As escolhas devem respeitar regras corporativas.

## RN-375

As escolhas devem respeitar limites financeiros.

## RN-376

As escolhas devem respeitar janelas de movimentação.

## RN-377

O sistema deve registrar histórico das escolhas.

## RN-378

Alterações devem possuir vigência definida.

## RN-379

O sistema deve permitir integração com plataformas flexíveis.

## RN-380

Toda movimentação deve ser auditável.

---

# Integrações com Fornecedores

## RN-381

O sistema deve permitir exportação de arquivos para operadoras.

## RN-382

O sistema deve permitir importação de arquivos de retorno.

## RN-383

O sistema deve permitir integração via API.

## RN-384

O sistema deve validar arquivos recebidos.

## RN-385

Arquivos inválidos devem gerar pendência.

## RN-386

Toda integração deve possuir log.

## RN-387

Falhas de integração devem gerar alerta.

## RN-388

O sistema deve permitir reprocessamento.

## RN-389

O sistema deve registrar histórico de transmissões.

## RN-390

Toda integração deve ser auditável.

---

# Descontos em Folha

## RN-391

Benefícios com participação financeira devem gerar descontos automáticos.

## RN-392

Os descontos devem respeitar vigência do benefício.

## RN-393

Os descontos devem respeitar regras da política corporativa.

## RN-394

Os descontos devem possuir rubrica específica.

## RN-395

O sistema deve permitir parcelamentos.

## RN-396

Parcelamentos devem manter memória de cálculo.

## RN-397

Diferenças devem gerar alertas.

## RN-398

Toda integração com folha deve ser rastreável.

## RN-399

Descontos cancelados devem manter histórico.

## RN-400

O sistema deve registrar auditoria completa dos descontos.

---

# Auditoria e Compliance

## RN-401

Toda concessão de benefício deve possuir histórico.

## RN-402

Toda suspensão deve possuir justificativa.

## RN-403

Toda exclusão deve possuir motivo registrado.

## RN-404

O sistema deve registrar usuário responsável por cada alteração.

## RN-405

O sistema deve registrar data e hora de cada alteração.

## RN-406

O sistema deve registrar logs de integrações.

## RN-407

O sistema deve permitir auditoria por colaborador.

## RN-408

O sistema deve permitir auditoria por benefício.

## RN-409

O sistema deve proteger dados pessoais conforme LGPD.

## RN-410

O sistema deve manter trilha completa de auditoria.
