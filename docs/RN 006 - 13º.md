# Sistema de RH para o Mercado Brasileiro

# Regras de Negócio (RN)

## Bloco 06 – Décimo Terceiro Salário

### Versão

1.0

### Objetivo

Este documento descreve as regras de negócio relacionadas ao cálculo, processamento, pagamento, encargos, integração com folha, eSocial e auditoria do décimo terceiro salário.

---

# Elegibilidade

## RN-501

Todo colaborador com vínculo empregatício ativo no ano-base deve ser avaliado para recebimento de décimo terceiro salário.

## RN-502

O sistema deve considerar a data de admissão para apuração dos avos de direito.

## RN-503

O sistema deve considerar a data de desligamento quando houver rescisão no ano-base.

## RN-504

O direito ao décimo terceiro deve ser calculado proporcionalmente aos meses trabalhados.

## RN-505

O sistema deve apurar avos de décimo terceiro por colaborador.

## RN-506

O sistema deve permitir consulta do saldo de avos apurados.

## RN-507

O sistema deve registrar memória de cálculo dos avos.

## RN-508

O sistema deve considerar afastamentos que impactem o direito ao décimo terceiro.

## RN-509

O sistema deve permitir parametrização de regras conforme tipo de vínculo.

## RN-510

Toda apuração deve possuir rastreabilidade.

---

# Primeira Parcela

## RN-511

O sistema deve permitir cálculo da primeira parcela do décimo terceiro.

## RN-512

A primeira parcela deve considerar os avos adquiridos até a competência de cálculo.

## RN-513

O sistema deve permitir pagamento da primeira parcela em folha específica.

## RN-514

O sistema deve permitir antecipação da primeira parcela junto às férias quando solicitado e permitido.

## RN-515

A antecipação junto às férias deve gerar registro específico.

## RN-516

O sistema deve impedir duplicidade de pagamento da primeira parcela.

## RN-517

O valor pago na primeira parcela deve ser armazenado para abatimento posterior.

## RN-518

O sistema deve gerar recibo ou demonstrativo da primeira parcela.

## RN-519

O sistema deve permitir conferência antes do fechamento.

## RN-520

Toda alteração manual deve possuir justificativa.

---

# Segunda Parcela

## RN-521

O sistema deve permitir cálculo da segunda parcela do décimo terceiro.

## RN-522

A segunda parcela deve considerar o valor total devido no ano-base.

## RN-523

O sistema deve abater automaticamente valores já pagos na primeira parcela.

## RN-524

O sistema deve considerar médias e variáveis aplicáveis.

## RN-525

O sistema deve calcular descontos legais aplicáveis.

## RN-526

O sistema deve gerar demonstrativo da segunda parcela.

## RN-527

O sistema deve permitir conferência dos valores antes do fechamento.

## RN-528

O sistema deve impedir pagamento duplicado da segunda parcela.

## RN-529

O sistema deve registrar data efetiva de pagamento.

## RN-530

Toda movimentação deve possuir trilha de auditoria.

---

# Médias e Verbas Variáveis

## RN-531

O sistema deve calcular médias de verbas variáveis quando aplicável.

## RN-532

Horas extras devem compor médias quando configuradas com incidência.

## RN-533

Comissões devem compor médias quando configuradas com incidência.

## RN-534

Adicional noturno deve compor médias quando aplicável.

## RN-535

Adicionais habituais devem compor médias conforme parametrização legal ou sindical.

## RN-536

O período de apuração das médias deve ser parametrizável.

## RN-537

O sistema deve apresentar memória de cálculo das médias.

## RN-538

Alterações em verbas variáveis devem permitir recálculo.

## RN-539

Médias recalculadas devem manter histórico.

## RN-540

Toda composição de média deve ser auditável.

---

# Encargos e Descontos

## RN-541

O sistema deve calcular INSS sobre décimo terceiro quando aplicável.

## RN-542

O sistema deve calcular FGTS sobre décimo terceiro quando aplicável.

## RN-543

O sistema deve calcular IRRF sobre décimo terceiro quando aplicável.

## RN-544

As bases de INSS, FGTS e IRRF devem ser armazenadas separadamente.

## RN-545

Rubricas de décimo terceiro devem possuir incidências próprias.

## RN-546

O sistema deve permitir parametrização das rubricas utilizadas no décimo terceiro.

## RN-547

O sistema deve gerar memória de cálculo dos encargos.

## RN-548

O sistema deve permitir conferência dos encargos antes do fechamento.

## RN-549

Diferenças identificadas devem gerar pendência de regularização.

## RN-550

Toda apuração de encargos deve possuir rastreabilidade.

---

# Afastamentos

## RN-551

O sistema deve considerar afastamentos na apuração do décimo terceiro.

## RN-552

Afastamentos previdenciários devem possuir tratamento específico.

## RN-553

Licença-maternidade deve possuir tratamento específico.

## RN-554

Suspensões contratuais devem impactar o cálculo conforme parametrização aplicável.

## RN-555

O sistema deve registrar quais afastamentos foram considerados no cálculo.

## RN-556

Alterações em afastamentos devem gerar alerta para recálculo.

## RN-557

O sistema deve manter histórico dos impactos por afastamento.

## RN-558

A apuração deve permitir auditoria por competência.

## RN-559

A apuração deve permitir auditoria por colaborador.

## RN-560

Toda regra aplicada ao afastamento deve ser rastreável.

---

# Integração com Rescisão

## RN-561

O sistema deve calcular décimo terceiro proporcional na rescisão quando aplicável.

## RN-562

O cálculo rescisório deve considerar avos adquiridos até a data de desligamento.

## RN-563

Valores já pagos de décimo terceiro devem ser abatidos na rescisão quando aplicável.

## RN-564

O sistema deve impedir duplicidade entre folha anual e rescisão.

## RN-565

O cálculo de décimo terceiro rescisório deve possuir memória detalhada.

## RN-566

A rescisão deve considerar médias de décimo terceiro quando aplicável.

## RN-567

O sistema deve integrar o valor calculado às verbas rescisórias.

## RN-568

Alterações na rescisão devem permitir recálculo do décimo terceiro proporcional.

## RN-569

O sistema deve registrar origem dos valores utilizados.

## RN-570

Todo cálculo rescisório deve ser auditável.

---

# Fechamento da Folha de Décimo Terceiro

## RN-571

A folha de décimo terceiro deve possuir status próprio.

## RN-572

O sistema deve permitir simulação antes do fechamento.

## RN-573

O sistema deve impedir fechamento com cálculos pendentes.

## RN-574

O sistema deve impedir fechamento com inconsistências críticas.

## RN-575

A folha fechada não poderá ser alterada sem reabertura formal.

## RN-576

A reabertura deve exigir justificativa.

## RN-577

A reabertura deve registrar usuário, data e motivo.

## RN-578

Após fechamento, os demonstrativos devem ser disponibilizados.

## RN-579

Após fechamento, os valores devem ser integrados aos encargos e obrigações.

## RN-580

Todo fechamento deve possuir trilha de auditoria.

---

# Integração com eSocial

## RN-581

O sistema deve gerar eventos de remuneração relacionados ao décimo terceiro quando aplicável.

## RN-582

O sistema deve gerar informações de pagamento relacionadas ao décimo terceiro quando aplicável.

## RN-583

Rubricas de décimo terceiro devem estar compatíveis com a tabela de rubricas do eSocial.

## RN-584

Eventos rejeitados pelo eSocial devem permanecer pendentes de correção.

## RN-585

O sistema deve registrar motivo da rejeição.

## RN-586

Eventos corrigidos devem permitir reenvio.

## RN-587

O recibo retornado pelo eSocial deve ser armazenado.

## RN-588

O sistema deve permitir conciliação entre folha calculada e totalizadores retornados.

## RN-589

Divergências devem gerar relatório de inconsistências.

## RN-590

Toda transmissão deve ser auditável.

---

# Relatórios

## RN-591

O sistema deve gerar relatório analítico de décimo terceiro.

## RN-592

O sistema deve gerar relatório sintético de décimo terceiro.

## RN-593

O sistema deve gerar relatório de avos por colaborador.

## RN-594

O sistema deve gerar relatório de primeira parcela.

## RN-595

O sistema deve gerar relatório de segunda parcela.

## RN-596

O sistema deve gerar relatório de encargos.

## RN-597

O sistema deve gerar relatório de médias utilizadas.

## RN-598

O sistema deve gerar relatório de diferenças e pendências.

## RN-599

O sistema deve permitir exportação dos relatórios.

## RN-600

Toda exportação deve respeitar permissões de acesso.

---

# Auditoria e Segurança

## RN-601

Todo cálculo de décimo terceiro deve possuir memória de cálculo.

## RN-602

Toda alteração manual deve exigir justificativa.

## RN-603

Toda alteração manual deve registrar usuário, data e hora.

## RN-604

O sistema deve controlar permissões para cálculo, conferência e fechamento.

## RN-605

Usuários sem permissão não podem visualizar valores de remuneração.

## RN-606

O sistema deve proteger dados pessoais e financeiros conforme LGPD.

## RN-607

O sistema deve registrar acessos aos demonstrativos.

## RN-608

O sistema deve registrar exportações de informações financeiras.

## RN-609

O sistema deve permitir auditoria por colaborador, competência e folha.

## RN-610

O sistema deve manter trilha completa de auditoria.
